# Project Walkthrough

Coderturtle.io is a public personal website built with Astro, Tailwind CSS, and MDX content, organized around real projects and their build logs rather than a blog. It is a Hekton factory-output project: the factory helps maintain and evolve it, but the site itself is the deliverable.

The adoption did not change app behavior. It added Hekton control-plane files so future agents can understand the project classification, repo location, public/private boundary, setup flow, risks, and next actions before editing.

The homepage revamp positions Coderturtle as the workshop voice of the wider ecosystem. Hekton is the factory, Agentic Tekton is the architecture/thought-leadership layer, Gremlins are the reusable product/lab direction, and Coderturtle is the builder/narrator who turns real work into notes, labs, workshops, and field reports.

The current front door follows the broader gateway pattern without naming the wider ecosystem on the first screen. `/` is a focused Coderturtle brand gateway with a single Enter action; `/enter/` is the interactive workshop homepage; `/ecosystem/` is a dedicated page for the wider Hekton/Agentic Tekton/Gremlins context (moved off `/enter/` on request so the homepage stays focused). The gateway's visual direction went through several revisions after the original dark/mono version (light-first illustration-led, then schematic/blueprint chrome) before landing, confirmed by the user, on **dark-first, "gateway to the future"**: `/` is a glowing portal (concentric rings, star field, a glowing framed portrait) with copy reframed around "the gateway to what's next"; dark is now the site's default theme everywhere (light remains fully functional via the toggle). `/enter/` is an interactive workshop scene: a master station (Coderturtle, click for a supervisor-themed quote) supervises three worker stations — Projects, Labs, and Workshops turtles, now rendered with real user-supplied illustrated portraits (`src/assets/turtles/`, routed through `astro:assets`) rather than the earlier reused-portrait placeholder; the worker art is a different illustration style (flat-vector, white background) from Coderturtle's own hand-drawn portrait, a contrast kept deliberately since it reads as "senior supervising junior specialists" rather than a mismatch. Projects links to the real `/projects/`, while Labs and Workshops are honest "still being built" buttons since those pages don't exist yet. `/ecosystem/` frames the same workshop as one cog meshing with Hekton/Agentic Tekton/Gremlins cogs; Agentic Tekton (`https://theagentictekton.com`) and Hekton (`https://hekton.theagentictekton.com`, confirmed by the user) are real outbound links — Gremlins still shows as "coming soon" since it has no confirmed public URL. `/about/` was rewritten with real first-person Coderturtle-voice content (replacing placeholder Lorem ipsum) and moved onto the shared `BaseLayout` design system.

The main thing to understand deeply is the deployment shape. The repository currently identifies as `coderturtle-blog` on GitHub, while the Hekton project identity is `coderturtle.io`. The app now builds for the domain root (`https://coderturtle.io/`) rather than the old GitHub Pages project path; the infra thread still owns S3, CloudFront, and workflow alignment.

The deployment review selected AWS static-site hosting as the production target. Terraform now lives under `infra/aws-static-site/`, the deploy manifest drives the Infrastructure Gremlin workflow, and `.github/workflows/deploy-aws-static-site.yml` is the manual-first deploy path. The current GitHub Pages workflow remains only as the legacy path until AWS infrastructure and deployment are verified.

The content model was rebuilt around **projects and build logs** instead of a blog, because a blog format-overlapped with Agentic Tekton's own editorial/thought-leadership site — projects-and-logs is a structurally different shape, not just a different skin. Two Astro content collections, `projects` and `logs` (`src/content.config.ts`), replace the old `blog` collection: a project lives at `/projects/<slug>/` with an in-page build-log timeline, and each log entry is individually addressable at `/projects/<slug>/log/<entry>/` for deep-linking by humans and, eventually, agents. `/blog/` now redirects to `/projects/`. The one real existing post ("Vibing with ChatGPT") was migrated as the first build-log entry of a new "Coderturtle.io" project rather than left behind, and doubles as the origin of a planned documentary narrative: the honest build log of this site being built with an AI agent in the loop. Verifying that migration surfaced and fixed two real pre-existing bugs — a silent Shiki/Mermaid rendering conflict (Mermaid diagrams were likely never actually rendering anywhere on the site) and a pile of dead legacy scaffold left over from the original Astro blog-starter template. The visual build-log component library is now built: `TerminalBlock`, `DiffBlock`, `Screenshot`, and `BeforeAfter` (`src/components/log/`) are MDX-embeddable and demonstrated with genuine content — real terminal transcripts, a real config diff, and real screenshots — in a second entry, "The dark reset," documenting today's actual design pivot. The agent-facing interface is also built: `/llms.txt`, `/projects/index.json`, and `/projects/<slug>.json` are all generated at build time from the `projects`/`logs` collections, so they can't drift from what the site shows; `.hekton/`'s internal control-plane data is intentionally never synced into them. The original four-phase content-architecture plan is now complete: the user confirmed the narrative stays documentary and separate from `hekton-field-journal`/`tekton-chronicle`'s gremlin fiction, and Phase 4 landed as a new `/about/` section stating that premise directly rather than leaving it implied by the build logs alone.

Coderturtle's voice is now documented, not just implied: `docs/voice-guide.md` pairs the cranky battle-scarred-engineer persona with a Coding Horror dry-humor reference style (blunt, self-deprecating, anecdote-first, no hype language). Agentic Tekton's own voice guide cites the same Coding Horror reference in a different register (executive/architect vs. hands-on engineer) — the guide addresses this directly rather than leaving it as an odd coincidence. The guide governs new content only; `0001-vibing-with-chatgpt.mdx` predates it and was deliberately left unrewritten, since retrofitting old entries to a later style would break the site's own promise that the build log isn't staged after the fact.

Skimmable material: the governance ledgers and repo-local `mind-palace/` mirror are traceability scaffolding. Agents can keep those current after material changes.
